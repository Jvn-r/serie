# DELI 11 - Proc Conn Integ 

## Objective
finally implement proc conn and move to an event driven process stream

## Learnings
- proc conn learnings from d8.md coming in clutch, went back and read all of the gathered info from that and used some of the references from there
- netlink comm flow : `socket -> bind -> register -> recieve message`
- registration uses an enum PROC_CN_MCAST_LISTEN, and unregistration uses PROC_CN_MCAST_IGNORE
- layering is `nlmsghdr->cn_msg->proc_event`
- recvmsg : counterpart to sendmsg, and returns a ssize_t with bytes processed or negative error values
- `sock_reci()` has local buffers, because it handles the raw data from netlink messages and we dont need to see the raw data elsewhere, we only need the extracted process deets, so using local ownership is simpler
- nl_pid, using gettid rather than getpid to avoid conflicts if a diff thread opens another nl sock, which can support multi threaded operations in the future, idk what thpse will be yet
- functions implemented, 
	- `sock_crea()` : create socket
	- `sock_send()` : send registration or unregistration netlink message
	- `sock_regi()` : sends enum PROC_CN_MCAST_LISTEN
	- `sock_unre()` : sends enum PROC_CN_MCAST_IGNORE
	- `sock_reci()` : recvmsg -> extract proc_event struct -> send to read_even
	- `read_even()` : read event->what to get fork, exec,exit etc, event and redirect to its respective function
		- for now just prints some of the details in the proc_event struc thats been revealed
		- `even_fork(struct proc_event *even);`
		- `even_exec(struct proc_event *even);`
		- `even_exit(struct proc_event *even);`
		- `even_comm(struct proc_event *even);`
		- `even_uid(struct proc_event *even);`
		- `even_gid(struct proc_event *even);`
- using `errno` and `strerror`, to show errors in a better ways
- using buffer overlaying to blueprint `nl_hdr` onto buffer, so that later a single iovec can be used to send the message easily
- various kinds of validation for a lot of things
	- nl msg types
	- proc conn routing 
	- core kernel validation
- saw the first successful live real time kernel event stream
- there are so many other things too like, all of the boilerplate meanings, how iovec works, all the new datatypes `ssize_t`, `uint8_t`, `void *` etc, `recvmsg `and `recvfrom` differences, exact parameters of fork, exec, exit, comm, etc. it was a very fun learning all of that, tho it was overwhelming at the start, but its comprehendable
- testing proc conn : `gcc -Wall -Wextra tests/main.c c_core/ser_proc_conn.c -o test ;./test`, other tests have been commented out
