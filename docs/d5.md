# DELI 5 - File Descriptors & /proc/<pid>/fd

## Objective
- Understand file descriptors and extract process file interactions using `/proc/<pid>/fd`

## Learnings
- file descriptors are the handles to files, sockets and pipes, this is how every process looks at files
- theyre all neatly stored in `/proc/<pid>/fd/`, and all the entries are indexed, and can be seen using `ls -l`
- `os.listdir` on `/fd` only gives the indexes
- the files from fd is found by their symlinks, which have to be resolved using `os.readlink()`
- fds have files, sockets, pipes and memory stuff
- need to filter and categorize all kinds of fds especially pipes that link multiple files, and files that are used bu multiple processes

## Outcome
fd's have been extracted, `{pid:{fd:target}}` is how theyre stored
