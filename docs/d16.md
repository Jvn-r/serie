# DELI 16 - comm, exec, UID and GID

## Objective
- finish the remaining proc connector events `comm, exec uid and gid`

## Learnings
- `comm`: strcpy the new name into proc->name
- `uid and gid`: update the real and effective uid gid values from the proc event
- `exec`: proc connector only provides the pid, and no more data
    - after an exec the process image changes, so i can call `prfs_look()`to refresh proc metadata and done
- first attempt at exec immediately segfaulted, valgrind pointed into `prfs_stat()`
    - fixed it, it was a proc_look that returned null, so the proc was null, so it had no mem assigned, added a null check in even_exec
- another valgrind run, found uninitialized reads inside `prfs_clean_fd()`
    - added more defensive checks into prfs_clean_fd
- fianlly another clean valgrind run, no leaks, no invalid reads

## Outcome
all remaining proc connector events are now handled.
- `even_exec()` refreshes proc metadata
- `even_comm()` updates process names
- `even_uid()` updates uid changes
- `even_gid()` updates gid changes
