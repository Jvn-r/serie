# DELI 17 - Periodic Logger

## Objective
- implement a periodic logger so the proc table automatically flushes every N seconds instead of only on SIGINT

## Learnings
- looked into Linux timer APIs,  old BSD `setitimer()` - `getitimer()` and the modern POSIX timer API 
- decided to use the POSIX timer API mainly coz i wanted to finally learn signal handling, apart from the simple SIGINT
- reused the signal handling system already made for `SIGINT` and added `SIGUSR1` trigger logger flush
- introduced another global flag
```c
volatile sig_atomic_t logg_run = 0;
```
- replaced old `signal()` registration with POSIX `sigaction()`
- logger flow now becomes
```
global logg_run = 0-> set up timer-> listen for new SIGUSR1-> signal handler sets logg_run = 1-> while running loop -> logg_parse_proc and logg_run = 0
```
- learnt how POSIX timers work from `man timer_create`
    - `timer_create()`
    - `timer_settime()`
    - `timer_delete()`
    - `struct sigevent`
    - `struct itimerspec`
- main loop simply checks the flag, signal handler only updates a flag, logger fires
- classic valgrind debugging sessions
    - `timer_settime()` returning `EINVAL`
        - forgot to zero initialise timer structs
        - also passed `timerid` incorrectly referenced into `timer_create()` and `timer_settime()`
	- uninitialised bytes passed into `timer_create()`
		- found it and made it NULL : `sev.sigev_value.sival_ptr = NULL;`
- had to make an early return in `recvmsg`, coz timer signals interrupt `recvmsg()`and it returns `EINTR` and returns -1, added an if to treat it as not a fault and just return and continue the event loop
- had to add a `-lrt` to gcc command to get the lib real time which is where all the new POSIX fns live

## Outcome
now we have periodic snapshots of the entire process table into NDJSON
