# DELI 15 - fd_paths Imple and Time Bug Fix 

## Objective
- fd paths implemented into json obj + wrong time fix

## Learnings
- fd's
	-  found `cJSON_CreateStringArray` inside `cJSON.h` which does exactly what i need for dealing with `**fd_paths` then make it into a string and add it to the `proc json`
	- saw a firefox process with 181 FDs, a ~9000 character proc object o7
	- no mem leaks
- time
	- narrowed down the possible problematic parts to  `clock_gettime  BOOTTIME or REALTIME`, `struct tm localtime_r`
	- everything else is timespec math and formatting
	- so went researching clocks and found out the kernel uses the MONOTONIC clock rather than the BOOTTIME,
		- boottime clock counts suspend time so its not close to what monotonic is, and my laptop hasnt shut down in a while
	- changed it and birth time works, then added some death time handling where if its not dead it wont have a death time
- updated `logg_proc_2_json` to add
```c
    if(p->death_time.tv_sec != 0)
        logg_time_reso(&p->death_time, dt_buff, buff_size);
    else
        dt_buff[0] = '\0';

    if(dt_buff[0] != '\0')
        cJSON_AddStringToObject(json, "death-time", dt_buff);
    else
        cJSON_AddStringToObject(json, "death-time", "not dead");

```

we now have clean and complete proc data, with fd's and correct wall clock timestamps
