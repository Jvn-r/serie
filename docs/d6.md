# DELI 6 - Refactoring to C 

## Objective
refactor code from prototyping python into C, which is what the actual system is gonna be on 

## Learnings
- new architecture
  - C : low level facing, actual data gathering from `/proc/`
  - python : analytics on data and future ML
- identifying core parts and refactoring it step by step
  - `/proc/` reader
  - snapshot comparison for lifecycle 
  - process relationshps
  - process tree 
  - fd's 
- `/proc/` reading
  - options to do this were `readdir()` or `scandir()`
    - `readdir()` : streams one by one, low memory, faster, no filtering or sorting
    - `scandir()` : loads whole dir into an array, uses malloc, more memory, allows for filtering and sorting
  - decided on `readdir()` because i dont need high level filtering, or sorting, `/proc` already sorts it numerically, just need to filter out any non numeric files, which is done with simple `if` statements
- need to preserve previous snapshot, for data, hence cant use destructive method to gather lifecycle of processes during the reading process
- have to do it similar to prototype with both prev and new snapshot comparison

## Outcome
working pid extraction with non numeric files filtered out, next steps on lifecycle are planned
