# DELI 3 - Discovering /stat and Getting Details From it

## Objective
extract and store process details

## Learnings
- `stat` is a single line of details prolly for other processes to read or sumn and `status` is a human readable structured repre of proc ddetails
- chose `stat` to be the source of info
- tokenizer for stat cane just be `.split()` because the proc name in stat is surrounded by brackets, so i used a Splice and Split Method(patent pending) to remove the brackets and tokenize and store it
- at this point teh scanner function which handled all of these and the previous tasks was getting cluttered and less reusable, thus 2 functions were born:
  - proc id reader : only reads ls /proc
  - proc details scanner : reads and tokenizes `stat` of all the pid's it receives
- at this point i realized that the loop was wildly ineffecient, as it read proc details of all PID's every iteration, which is like 300-400 file read ops, instead, we can read only the birthed process details and add it to a global dict carrying all proc details
- added this to compare fn, where after the birth's are gotten we just call proc details scanner and send the set of birthed pids
- one small hitch was that it only read the births, BUT, when i start the program, it didnt recognixze the initial PID set as births, so ot only didnt read the first set of PID's which were most of them, so just added a compare out of the loop passing set of pid and a null set, so that when program starts it will consider all the pids it sees as births, thus reads all the process details, and later we just append any new process details to it, and DEATHS were just deleted from the global dict 
- one more note, from `stat` we extract `name, state, ppid`, so now we can do PARENT - CHILD relationships next
