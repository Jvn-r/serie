# DELI 9 - Proc Hash table Implementation

## Objective
- build the basic process storage and lookup layer

## Learnings
- hash table and proc tree distinction
    - hash table = lookup + storage
    - proc tree = relationships
- bucket collisions, where multiple procs in one bucket, haneled using a linked list chain
- use a wrapper proc_buck_node for hashing for now, coud move to an embedded hash_next variable inside the proc struct
    - coz rn we have 2x allocations, one for proc and one for proc_buck_node
    - rn it is simpler implementation without disturbing the actual process metadata,
- heap alloc and stack alloc intricacies
- memory ownership inside hash 
```
table  
|  
|--> buckets array  
|   |
|   |--> bucket node  
|   |   |--> proc  
|   |   |--> proc
|   |--> bucket node  
|   |   |-- proc  
|   ...
```
